# 智能植物养护系统

> 本项目为比赛参赛作品，赛后整理开源。

基于 ESP32-S3 的自动化植物管理系统，集成多种传感器、摄像头和云端服务，实现根据环境数据自动浇水、施肥、补光，并通过图像识别评估植物健康状态。

## 系统架构

```
硬件层：  ESP32-S3 + 传感器阵列 + 执行器（水泵/补光灯/蜂鸣器） + OV2640摄像头
控制层：  Arduino 固件（传感器采集、智能决策、图像处理、网络通信）
云端层：  Node.js + Express + SQLite + RESTful API
展示层：  响应式 Web 管理界面（数据可视化、远程配置、历史查询）
```

## 硬件组成

| 模块 | 型号/引脚 | 说明 |
|------|----------|------|
| 主控 | ESP32-S3 | 不要用带 R8 后缀的型号，部分无法驱动摄像头 |
| 温湿度 | AM2301 (GPIO36) | 数字温湿度传感器 |
| 光照 | 模拟传感器 (GPIO3) | 模拟输入，12位ADC |
| 土壤湿度 | 模拟传感器 (GPIO4) | 模拟输入，12位ADC |
| EC传感器 | RS485 (GPIO16/17) | UART通信，GPIO38控制方向 |
| 摄像头 | OV2640 | 专用并口引脚 |
| 显示屏 | SSD1306 OLED (GPIO7/18) | I2C接口，128x64 |
| 浇水泵 | PWM (GPIO40) | 5V蠕动泵 |
| 肥料A泵 | PWM (GPIO37) | 5V蠕动泵 |
| 肥料B泵 | PWM (GPIO12) | 5V蠕动泵 |
| 补光灯 | PWM (GPIO39) | LED植物补光灯 |
| 蜂鸣器 | GPIO21 | 有源蜂鸣器 |

## 功能列表

1. **多传感器采集** — 每15分钟读取温湿度、光照、土壤湿度、EC值
2. **智能水肥决策** — 基于土壤湿度和EC值，结合温度、光照补偿，动态计算浇水与施肥量
3. **生长阶段识别** — 根据植物日龄、健康度、EC值自动判断幼苗期/生长期/开花期/结果期
4. **叶片健康评估** — 摄像头拍照后通过HSV颜色空间分析叶片健康度，记录30天趋势
5. **自动补光** — 模拟日升日落，5:00-20:00亮度渐变，拍照时强制全亮
6. **远程配置** — 通过网页修改目标湿度、EC阈值、冷却时间等参数，ESP32每小时同步
7. **云端存储** — 传感器数据与照片上传至服务器，支持历史查询与统计

## 软件依赖

### ESP32端

- Arduino IDE 3.x
- 开发板：ESP32S3 Dev Module
- 库文件：DHT sensor library、U8g2、ArduinoJson、ESP32-Camera

### 服务器端

- Node.js 18+
- 依赖包：express、multer、sqlite3、cors

```bash
cd 网页端源码/
npm install
npm start
```

## 部署步骤

### 1. 硬件组装

按照引脚定义连接传感器、执行器和摄像头。推荐先使用面包板验证，确认无误后使用PCB焊接。

### 2. 固件烧录

1. 修改 `esp32wsdecision.ino` 中的 WiFi 名称、密码和服务器 IP 地址
2. 使用 Arduino IDE 选择 ESP32S3 Dev Module 开发板
3. 编译并烧录

### 3. 服务器搭建

```bash
cd plant-system/
npm install
npm start         # 开发环境
# 生产环境建议使用 pm2:
pm2 start server.js --name plant-system
```

### 4. 访问管理界面

浏览器打开 `http://服务器IP:3000`

## 关于调试代码

为方便测试，固件中保留了两处调试代码：

1. **上电自检**（setup 函数末尾） — 启动后依次测试水泵、补光灯、蜂鸣器，确认所有执行器工作正常。正式部署时可删除或注释。

2. **拍照间隔**（loop 函数中） — 当前设置为10秒拍照一次用于快速验证。正式部署时需改为24小时间隔（代码中已注释，搜索"10秒拍照测试"定位）。
